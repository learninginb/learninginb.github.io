java进阶学习笔记

发表于 2018-03-30

面向对象

1、继承，重载，重写，多态

继承：常见好用，减少代码量，除了变量，方法也能够继承
重写：继承的子类的方法可以重写，自动先调用子类的方法
重构：一样的函数名，可以不一样的参数或者顺序，是不同的方法
多态：继承的父类子类可以转型，并调用子类的方法。调用父类的方法要借助super关键字。调用父类子类都有的方法时，会有限调用子类重写的方法。

2、抽象类

用关键字abstract来定义抽象类，抽象类无法被实例化。只能先用一个类继承这个抽象类，再实例化这个类。static类不能为抽象类

  1. 抽象类不能被实例化(初学者很容易犯的错)，如果被实例化，就会报错，编译无法通过。只有抽象类的非抽象子类可以创建对象。
  2. 抽象类中不一定包含抽象方法，但是有抽象方法的类必定是抽象类。
  3. 抽象类中的抽象方法只是声明，不包含方法体，就是不给出方法的具体实现也就是方法的具体功能。
  4. 构造方法，类方法（用static修饰的方法）不能声明为抽象方法。
  5. 抽象类的子类必须给出抽象类中的抽象方法的具体实现，除非该子类也是抽象类。

封装的优点

  1. 良好的封装能够减少耦合。
  2. 类内部的结构可以自由修改。
  3. 可以对成员变量进行更精确的控制。
  4. 隐藏信息，实现细节。

4、接口 Interface

接口可多继承的意思是:
public class Extends Interface0, Interface1, interface……
接口中的成员变量只能是 public static final 类型的。不写就默认是这个类型的，写了其他的就会报错。

接口的实现：

public class XXX implements 接口名称[, 其他接口名称, 其他接口名称..., ...]

1）枚举：

也有构造函数，定义全局常量，非常方便
枚举定义了一个叫nextElement 的方法，该方法用来得到一个包含多元素的数据结构的下一个元素。

2）位集合（BitSet）：

BitSet bits1 = new BitSet(size);

位集合类实现了一组可以单独设置和清除的位或标志。
用来逻辑运算的，定义了很多逻辑运算方法，但我没用过，应该是很高效的东西。

3）向量（Vector）：

Vector v = new Vector();

向量（Vector）类和传统数组非常相似，但是Vector的大小能根据需要动态的变化。
和数组一样，Vector对象的元素也能通过索引访问。
使用Vector类最主要的好处就是在创建对象的时候不必给对象指定大小，它的大小会根据需要动态的变化。

科学计算应该会很有用 向量文档

4）栈（Stack）：

栈是Vector的一个子类，它实现了一个标准的后进先出的栈。

13、Java 多线程编程

这个玩意厉害了，大中型项目一定会用到多线程。
能够更好的利用CPU。

创建线程的三种方法：
1、通过实现 Runnable 接口；
2、通过继承 Thread 类本身；
3、通过 Callable 和 Future 创建线程。
有效利用多线程的关键是理解程序是并发执行而不是串行执行的。例如：程序中有两个子系统需要并发执行，这时候就需要利用多线程编程。
通过对多线程的使用，可以编写出非常高效的程序。不过请注意，如果你创建太多的线程，程序执行的效率实际上是降低了，而不是提升了。
请记住，上下文的切换开销也很重要，如果你创建了太多的线程，CPU 花费在上下文的切换的时间将多于执行程序的时间！

14、Java 文档注释

/*** 这个类的功能，以及怎么实现的
* @author xuqi
* @version 1.2
* @exception 异常
* @param vo 解释变量
* @return 解释返回值
* @throws 也是异常
* 还有很多，但常见的就这几个了
*/

各位大神，写代码一定要写文档和注释，，，方便他人也方便自己
